🎯 סקירה כללית

מה נבנה?

נקים אפליקציית Single-Tenant ב-Microsoft Entra ID עם ההרשאות והתצורות הנדרשות לאימות משתמשים ומשיכת נתונים ממערכת Entra.

✨יתרונות הגישה המאוחדת
  • ניהול פשוט: אפליקציה אחת במקום מספר אפליקציות
  • אבטחה משופרת: Client Secret אחד לנהל ולחדש
  • תחזוקה קלה: עדכונים והגדרות במקום מרכזי אחד
  • עלות נמוכה: פחות מורכבות = פחות זמן ניהול

מה נדרש?

🏢 Account Type

Single-Tenant

גישה למשתמשים בארגון שלכם בלבד

🔗 Redirect URIs

2 URIs

Web + Public Client

🔑 Permissions

Delegated

User.Read.All + GroupMember.Read.All

✅ Public Flows

Required

חובה להפעלה

🛠️ דרישות טכניות

רכיב ערך נדרש הערות
Supported Account Types Single-Tenant Accounts in this organizational directory only
Redirect URI #1 Web https://your-app-name.streamlit.app (או כתובת הממשק שלכם)
Redirect URI #2 Public client/native אופציונלי - לפי הגדרות SafeQ Cloud
Delegated Permissions User.Read.All
GroupMember.Read.All		 שתי הרשאות חובה + Admin Consent
Allow Public Client Flows Yes קריטי! חובה להפעלה
Client Secret 24 חודשים יש לחדש לפני התפוגה
⚠️חשוב לדעת

Public Client Flows הוא הגדרה קריטית שבלעדיה Service Account Sync לא יעבוד. ודאו שהגדרה זו מופעלת (Yes) בחלק Authentication של האפליקציה.

📚 מדריך שלב אחר שלב

1 כניסה ל-Azure Portal

פעולה:

  1. היכנסו ל-Azure Portal: https://portal.azure.com
  2. התחברו עם חשבון בעל הרשאות Global Administrator או Application Administrator
  3. חפשו "Microsoft Entra ID" בסרגל החיפוש העליון
  4. לחצו על "Microsoft Entra ID" מהתוצאות
2 יצירת App Registration חדש

פעולה:

  1. בתפריט הימני, לחצו על "App registrations"
  2. לחצו על הכפתור "+ New registration" בחלק העליון
  3. מלאו את הפרטים הבאים:
Name: SafeQ Cloud - Unified App (או כל שם אחר שמתאים לארגון שלכם) Supported account types: ☑ Accounts in this organizational directory only (Single tenant) Redirect URI (optional): השאירו ריק בשלב זה - נוסיף מאוחר יותר

4. לחצו על הכפתור "Register"

✓בוצע בהצלחה

האפליקציה נוצרה! עכשיו נעבור להגדרות מתקדמות.

3 שמירת מזהי האפליקציה

פעולה:

אחרי שהאפליקציה נוצרה, תועברו לעמוד ה-Overview שלה. העתיקו ושמרו את הערכים הבאים במקום מאובטח:

שדה איפה למצוא דוגמה
Application (client) ID Overview → Application (client) ID 12345678-1234-1234-1234-123456789012
Directory (tenant) ID Overview → Directory (tenant) ID 87654321-4321-4321-4321-210987654321
⚠️חשוב

שמרו ערכים אלו במקום מאובטח (Password Manager או קובץ מוצפן). תצטרכו אותם בהמשך!

4 הוספת Redirect URIs

פעולה:

  1. בתפריט הימני של האפליקציה, לחצו על "Authentication"
  2. בחלק "Platform configurations", לחצו "+ Add a platform"
  3. בחרו "Web"
  4. הזינו את ה-Redirect URI של הממשק שלכם:
Redirect URIs: https://your-app-name.streamlit.app (החלף את your-app-name בשם האפליקציה שלך)

5. לחצו "Configure"

6. (אופציונלי) אם SafeQ Cloud דורש Redirect URI נוסף, לחצו שוב "+ Add a platform", בחרו "Mobile and desktop applications", והזינו את הכתובת הנדרשת.

7. לחצו "Save" בחלק העליון של הדף

💡טיפ

Azure מאפשר להוסיף מספר Redirect URIs לאותה אפליקציה. זה מאפשר לכם להשתמש באפליקציה אחת למספר מטרות!

5 הפעלת Public Client Flows

פעולה:

אתם כבר בעמוד Authentication, גללו למטה ל:

  1. מצאו את החלק "Advanced settings"
  2. תחת "Allow public client flows"
  3. הפכו את הערך ל-"Yes"
  4. לחצו "Save" בחלק העליון
🚨קריטי!

Allow public client flows חייב להיות מופעל (Yes) כדי ש-Service Account Sync יעבוד. זוהי דרישה קריטית של SafeQ Cloud!

6 הגדרת הרשאות API

פעולה:

  1. בתפריט הימני, לחצו על "API permissions"
  2. תראו הרשאה ברירת מחדל: User.Read - מחקו אותה (לחצו על ... ← Delete permission)
  3. לחצו על "+ Add a permission"
  4. בחרו "Microsoft Graph" מהרשימה
  5. בחרו "Delegated permissions"
  6. חפשו והוסיפו את ההרשאות הבאות:
☑ User.Read.All קריאת פרופילים מלאים של משתמשים ☑ GroupMember.Read.All קריאת חברות בקבוצות

7. לחצו "Add permissions"

8. כעת לחצו על הכפתור "Grant admin consent for [Your Organization]"

9. אשרו על ידי לחיצה על "Yes"

✓הרשאות הוגדרו

אתם אמורים לראות סימן ✓ ירוק בעמודת "Status" ליד כל הרשאה.

📌למה ההרשאות האלה?
  • User.Read.All: מאפשר קריאת מידע מלא על משתמשים (שם, אימייל, מחלקה, וכו')
  • GroupMember.Read.All: מאפשר לדעת לאילו קבוצות משתמש שייך (לצורך הרשאות)
7 יצירת Client Secret

פעולה:

  1. בתפריט הימני, לחצו על "Certificates & secrets"
  2. לחצו על הטאב "Client secrets"
  3. לחצו "+ New client secret"
  4. מלאו את הפרטים:
Description: SafeQ Manager & Service Account Secret Expires: 24 months (המלצה)

5. לחצו "Add"

🔒חשוב מאוד!

העתיקו את ה-Value (הסוד) מיד! הוא מוצג רק פעם אחת ולא ניתן לשחזר אותו מאוחר יותר.

אם לא העתקתם - תצטרכו ליצור Client Secret חדש.

6. שמרו את ה-Value (לא את ה-Secret ID!) במקום מאובטח

🗓️תזכורת

Client Secret פג תוקף אחרי 24 חודשים. הגדירו תזכורת ביומן לחידוש הסוד 2-3 שבועות לפני התפוגה!

8 סיכום ואימות

בדיקת תקינות:

ודאו שיש לכם את כל הפרטים הבאים:

פרמטר תיאור סטטוס
Application (client) ID מזהה האפליקציה ✅
Directory (tenant) ID מזהה הטננט ✅
Client Secret (Value) הסוד של האפליקציה ✅
Redirect URI כתובת הממשק ✅
🎉סיימתם!

האפליקציה ב-Entra ID מוכנה לשימוש!
כעת תוכלו להעביר את הפרטים הללו לצוות הטכני להגדרה במערכת SafeQ Cloud Manager ובממשק ניהול SafeQ Cloud.

📊 טבלת סיכום - מה הגדרנו?

✅ Checklist סופי

ודאו שכל הפריטים הבאים בוצעו לפני סגירת המדריך:

  • נוצרה App Registration חדשה ב-Microsoft Entra ID
  • נבחר Account Type: Single-Tenant
  • נוספו Redirect URIs (לפחות אחד מסוג Web)
  • הופעל Allow Public Client Flows = Yes
  • נוספו הרשאות: User.Read.All + GroupMember.Read.All
  • ניתן Admin Consent להרשאות
  • נוצר Client Secret בתוקף של 24 חודשים
  • הועתקו ונשמרו: Application ID, Tenant ID, Client Secret
  • הפרטים הועברו לצוות הטכני להגדרה במערכת
  • נקבעה תזכורת לחידוש Client Secret לפני התפוגה

❓ שאלות ותשובות

❓ האם Public Client Flows מסכן את האבטחה?

תשובה:

לא, כל עוד אתם משתמשים גם ב-Client Secret (שאתם משתמשים). הממשק שלכם משתמש ב-Authorization Code Flow עם PKCE + Client Secret, שזה מאובטח מאוד.

Public Client Flows נדרש ספציפית ל-Service Account כדי לאפשר אימות דרך SafeQ Cloud.

❓ מה קורה אם שכחתי להעתיק את ה-Client Secret?

תשובה:

אין בעיה! פשוט צרו Client Secret חדש:

  1. חזרו ל-Certificates & secrets
  2. צרו Secret חדש
  3. העתיקו את ה-Value הפעם
  4. (אופציונלי) מחקו את הסוד הישן אם אינכם זקוקים לו
❓ האם אפשר לשנות את שם האפליקציה אחרי היצירה?

תשובה:

כן! פשוט עברו ל-Overview → לחצו על "Display name" ושנו את השם.
שימו לב ש-Application ID ו-Tenant ID לא משתנים.

❓ כמה זמן לוקח עד שההגדרות נכנסות לתוקף?

תשובה:

שינויים ב-Entra ID בדרך כלל נכנסים לתוקף מיידי.
במקרים נדירים, זה יכול לקחת עד 5-10 דקות עד שהשינויים מתפשטים לכל השרתים של Microsoft.

❓ מה קורה כאשר ה-Client Secret פג תוקף?

תשובה:

כאשר Client Secret פג תוקף, המערכת תפסיק לעבוד ותקבלו שגיאות אימות.

למניעת הפסקה:

  • הגדירו תזכורת ביומן 2-3 שבועות לפני התפוגה
  • צרו Client Secret חדש מראש
  • עדכנו את המערכת עם הסוד החדש
  • מחקו את הסוד הישן רק אחרי שהמערכת עובדת עם החדש
❓ האם צריך להוסיף הרשאות Application במקום Delegated?

תשובה:

לא. במקרה שלנו, כל ההרשאות הן מסוג Delegated בלבד.

Delegated = הרשאות בשם משתמש מחובר
Application = הרשאות ברקע ללא משתמש (לא נדרש כאן)

❓ איך אני יודע שההגדרה הצליחה?

תשובה:

אחרי שהעברתם את הפרטים לצוות הטכני, הם יבצעו בדיקת אינטגרציה:

  1. ניסיון כניסה לממשק SafeQ Cloud Manager
  2. אימות שמשתמשי Entra יכולים להתחבר
  3. בדיקת Service Account Sync ב-SafeQ Cloud
  4. אימות שמשיכת נתונים מ-Entra עובדת

אם כל הבדיקות עברו - ההגדרה הצליחה! 🎉
