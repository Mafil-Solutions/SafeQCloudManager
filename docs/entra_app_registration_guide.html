🎯 סקירה כללית

מה נבנה?

נקים אפליקציית Single-Tenant ב-Microsoft Entra ID עם ההרשאות והתצורות הנדרשות לאימות משתמשים ומשיכת נתונים ממערכת Entra.

✨יתרונות הגישה המאוחדת
  • ניהול פשוט: אפליקציה אחת במקום מספר אפליקציות
  • אבטחה משופרת: Client Secret אחד לנהל ולחדש
  • תחזוקה קלה: עדכונים והגדרות במקום מרכזי אחד
  • עלות נמוכה: פחות מורכבות = פחות זמן ניהול

✓

⚠️חשוב לדעת

Public Client Flows הוא הגדרה קריטית שבלעדיה Service Account Sync לא יעבוד. ודאו שהגדרה זו מופעלת (Yes) בחלק Authentication של האפליקציה.

📚 מדריך שלב אחר שלב

1 כניסה ל-Azure Portal

פעולה:

  1. היכנסו ל-Azure Portal: https://portal.azure.com
  2. התחברו עם חשבון בעל הרשאות Global Administrator או Application Administrator
  3. חפשו "Microsoft Entra ID" בסרגל החיפוש העליון
  4. לחצו על "Microsoft Entra ID" מהתוצאות
2 יצירת App Registration חדש

פעולה:

  1. בתפריט הימני, לחצו על "App registrations"
  2. לחצו על הכפתור "+ New registration" בחלק העליון
  3. מלאו את הפרטים הבאים:
Name: SafeQ Cloud - Unified App (או כל שם אחר שמתאים לארגון שלכם) Supported account types: ☑ Accounts in this organizational directory only (Single tenant) Redirect URI (optional): השאירו ריק בשלב זה - נוסיף מאוחר יותר

4. לחצו על הכפתור "Register"

✓בוצע בהצלחה

האפליקציה נוצרה! עכשיו נעבור להגדרות מתקדמות.

3 שמירת מזהי האפליקציה

פעולה:

אחרי שהאפליקציה נוצרה, תועברו לעמוד ה-Overview שלה. העתיקו ושמרו את הערכים הבאים במקום מאובטח:

שדה איפה למצוא דוגמה
Application (client) ID Overview → Application (client) ID 12345678-1234-1234-1234-123456789012
Directory (tenant) ID Overview → Directory (tenant) ID 87654321-4321-4321-4321-210987654321
⚠️חשוב

שמרו ערכים אלו במקום מאובטח (Password Manager או קובץ מוצפן). תצטרכו אותם בהמשך!

4 הוספת Redirect URI

פעולה:

  1. בתפריט הימני של האפליקציה, לחצו על "Authentication"
  2. בחלק "Platform configurations", לחצו "+ Add a platform"
  3. בחרו "Web"
  4. הזינו את ה-Redirect URI של הממשק:
Redirect URIs: https://your-app-name.streamlit.app (החלף את your-app-name בשם האפליקציה שלך)

5. לחצו "Configure"

6. לחצו "Save" בחלק העליון של הדף

💡טיפ

אם יש צורך בעתיד, ניתן להוסיף Redirect URIs נוספים לאותה אפליקציה!

5 הפעלת Public Client Flows

פעולה:

אתם כבר בעמוד Authentication, גללו למטה ל:

  1. מצאו את החלק "Advanced settings"
  2. תחת "Allow public client flows"
  3. הפכו את הערך ל-"Yes"
  4. לחצו "Save" בחלק העליון
🚨קריטי!

Allow public client flows חייב להיות מופעל (Yes) כדי ש-Service Account Sync יעבוד. זוהי דרישה קריטית של SafeQ Cloud!

6 הגדרת הרשאות API

פעולה:

  1. בתפריט הימני, לחצו על "API permissions"
  2. תראו הרשאה ברירת מחדל: User.Read - אל תמחקו אותה!
  3. לחצו על "+ Add a permission"
  4. בחרו "Microsoft Graph" מהרשימה
  5. בחרו "Delegated permissions"
  6. חפשו והוסיפו את ההרשאות הבאות (בנוסף ל-User.Read הקיים):
☑ User.Read.All קריאת פרופילים מלאים של כל המשתמשים ☑ GroupMember.Read.All קריאת חברות בקבוצות ☑ Directory.Read.All קריאת מידע מהספרייה

7. לחצו "Add permissions"

8. כעת לחצו על הכפתור "Grant admin consent for [Your Organization]"

9. אשרו על ידי לחיצה על "Yes"

✓הרשאות הוגדרו

אתם אמורים לראות סימן ✓ ירוק בעמודת "Status" ליד כל הרשאה.

📌סיכום ההרשאות

רשימת ההרשאות הסופית שאמורה להיות לכם:

  • User.Read - הרשאה ברירת מחדל
  • User.Read.All - קריאת כל המשתמשים
  • GroupMember.Read.All - קריאת קבוצות
  • Directory.Read.All - קריאת ספרייה
7 יצירת Client Secret

פעולה:

  1. בתפריט הימני, לחצו על "Certificates & secrets"
  2. לחצו על הטאב "Client secrets"
  3. לחצו "+ New client secret"
  4. מלאו את הפרטים:
Description: SafeQ Cloud Secret Expires: 24 months (המלצה)

5. לחצו "Add"

🔒חשוב מאוד!

העתיקו את ה-Value (הסוד) מיד! הוא מוצג רק פעם אחת ולא ניתן לשחזר אותו מאוחר יותר.

אם לא העתקתם - תצטרכו ליצור Client Secret חדש.

6. שמרו את ה-Value (לא את ה-Secret ID!) במקום מאובטח

🗓️תזכורת

Client Secret פג תוקף אחרי 24 חודשים. הגדירו תזכורת ביומן לחידוש הסוד 2-3 שבועות לפני התפוגה!

8 סיכום ואימות

בדיקת תקינות:

ודאו שיש לכם את כל הפרטים הבאים:

פרמטר תיאור סטטוס
Application (client) ID מזהה האפליקציה ✅
Directory (tenant) ID מזהה הטננט ✅
Client Secret (Value) הסוד של האפליקציה ✅
Redirect URI כתובת הממשק ✅
🎉סיימתם!

האפליקציה ב-Entra ID מוכנה לשימוש!
כעת תוכלו להעביר את הפרטים הללו לצוות הטכני להגדרה במערכת SafeQ Cloud.
